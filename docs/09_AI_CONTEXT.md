# AI Context

> **This document provides essential context for AI coding assistants working on Military Companion.**

---

# Project Overview

Military Companion is a mobile-first platform that helps Korean soldiers make the most of their military service.

It is not simply a leave tracker or information database.

The product transforms military knowledge into personalized guidance throughout a soldier's journey.

Every engineering decision should support this mission.

---

# Product Mission

Help every Korean soldier get the most out of military service.

---

# Core Philosophy

Military Companion should feel like the senior every soldier wishes they had.

The application should:

* Reduce uncertainty.
* Save time.
* Surface opportunities.
* Build confidence.
* Support soldiers throughout their journey.

---

# Primary User

The initial target user is an active-duty Republic of Korea soldier.

The MVP focuses on solving problems during active military service.

---

# MVP Focus

The first version should prioritize:

* Personalized military journey
* Timeline and milestones
* Leave planning
* Promotion tracking
* Benefits discovery
* AI guidance
* Career preparation

Avoid adding unrelated features.

---

# Engineering Principles

When implementing features:

* Prioritize simplicity.
* Prefer readability over cleverness.
* Write maintainable code.
* Keep components modular.
* Minimize unnecessary dependencies.
* Follow existing project conventions.

---

# AI Expectations

When generating code:

* Do not invent product requirements.
* Follow existing documentation.
* Ask for clarification if requirements are ambiguous.
* Preserve backward compatibility whenever possible.
* Explain architectural trade-offs before major changes.

---

# User Experience

Every screen should answer a clear user question.

Avoid overwhelming users with unnecessary information.

Personalization is preferred over generic content.

---

# Technology Philosophy

Choose technologies that:

* Improve maintainability.
* Scale with the product.
* Are well supported by AI coding tools.
* Reduce operational complexity.

Do not introduce technologies solely because they are new or popular.

---

# Long-Term Vision

Military Companion aims to become the trusted digital companion for every Korean soldier.

Future partnerships, expanded services, and new products should build upon the existing architecture rather than replacing it.

---

# Working With This Repository

Before making significant changes:

1. Read the project documentation.
2. Understand the user's journey.
3. Identify the problem being solved.
4. Implement the simplest solution that satisfies the requirement.
5. Document important decisions.

The goal is not to generate the most code.

The goal is to generate the right code.

---

# Final Reminder

Every line of code should contribute toward one objective:

> **Helping soldiers make the most of their military service.**
